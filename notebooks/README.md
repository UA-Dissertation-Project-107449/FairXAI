# Notebooks

Exploratory and dissertation-support notebooks for FairXAI. Notebooks consume
pipeline artifacts; production logic should live in `src/` or `scripts/`.

See [../docs/README.md](../docs/README.md) for the full docs index.

## Expected Inputs

Profiling artifacts:

```text
output/<pipeline>/runs/<run_id>/profiling/
output/<pipeline>/profiling/        # standalone profiling fallback
```

Files:

- `<dataset>_data_profile.json`
- `<dataset>_complexity.json`
- `dataset_comparison.json`

Latest-run lookup uses `output/<pipeline>/latest_run/` when available, then
falls back to standalone profiling outputs.

## Profile JSON Shape

Each `<dataset>_data_profile.json` includes:

- `dataset_name`
- `basic_stats`
- `sensitive_attr_distribution`
- `target_distribution`
- `group_statistics`
- `representation_balance`
- `label_imbalance_by_group`
- `missing_value_analysis`
- `complexity_metrics`
- `group_complexity_metrics`
- `intersection_complexity_metrics`

The lighter `<dataset>_complexity.json` contains the `complexity_metrics` block only (useful for API consumption).

## Outputs

| Path | Purpose |
|------|---------|
| `notebooks/tables/<pipeline>/` | Exported notebook tables |
| `notebooks/figures/<pipeline>/` | Exported exploratory figures |

Dissertation batch figures generated by scripts belong under
`output/cardiac/studies/dissertation_figures/<run_id>/`, not this folder.

## Helpers

Use `fairxai.notebook_utils` for path resolution, dataset loading, schema-aware
age/sex labels, palettes, and labeling utilities.
